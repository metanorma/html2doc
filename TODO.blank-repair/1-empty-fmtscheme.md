# Bug 1: Theme fmtScheme has empty required lists

## Symptom
Word reports "Summary Info 1 is corrupted" on DOCX files created from scratch by Uniword.

## Root Cause
`word/theme/theme1.xml` has an empty `<a:fmtScheme>`:

```xml
<a:fmtScheme>
  <a:fillStyleLst/>
  <a:lnStyleLst/>
  <a:effectStyleLst/>
  <a:bgFillStyleLst/>
</a:fmtScheme>
```

OOXML requires minimum content in each list:
- `fillStyleLst` — at least 2 fill elements
- `lnStyleLst` — at least 3 line elements
- `effectStyleLst` — at least 3 effectStyle elements
- `bgFillStyleLst` — at least 2 fill elements

Word-generated themes have rich content in all four lists (see `rice_roundtrip6.docx` theme for reference).

## Evidence
- `rice_roundtrip6.docx` (works) has full fmtScheme content → opens in Word
- `minimal_fresh.docx` (broken) has empty fmtScheme → "Summary Info 1 corrupt"
- Swap test: rice base + minimal app.xml works (rice theme intact)
- Swap test: minimal base + rice app.xml fails (minimal theme still broken)
- Word repair replaces the entire theme with a full default theme (1659B → 8717B)

## Fix Location
The theme template is generated by Uniword's reconciler. Check where the default theme XML comes from:

1. `reconciler.rb` → `reconcile_theme` method
2. Theme model class (`Drawingml::Theme`) — check if `fmtScheme` attribute defaults to empty
3. Any template XML file used as the theme source

## Fix Strategy
Replace the empty fmtScheme with the standard Office theme fmtScheme content (from `rice.docx` or a known-good Word theme).
